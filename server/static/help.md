# Help

Use these pages as the starting point for the UI and the implementation details behind it.

- [Creations dashboard](../creations) - fixed population report with local aggregation and live evaluation updates.
- [Observability](../observability) - baby Grafana view for aggregated worker snapshots and time-series inspection.
- [Recent changes worker](recent-changes.md) - explains how recent changes updates the cache and fills creation metadata.
- [EntityData worker](entitydata.md) - explains how EntityData picks deletion-log work, subscribed work, and stale items.
- [Inlinks worker](inlinks.md) - explains how inlinks selects subscribed, cache-only, and refresh work.
- [Badge](badge.md) - explains what each color and region of the notability badge means.
- [Detectors](detectors.md) - describes the evidence sources and how each criterion is judged.
- [Data flow and caches](data-flow.md) - describes the worker pipeline, cache layout, and update paths.

If you are looking at a report page, use the badge link in the header to jump back to the current item report.
